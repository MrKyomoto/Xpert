# 技能：三级冲突仲裁

识别冲突性质，依证据优先级作出最小风险决策。

## 第一级：识别关系

- 将提案判为无分歧、表面分歧、可组合分歧或真分歧。
- 对相同问题与方向去重；对不同层级的互补建议使用 `merged`。
- 仅当两个建议不能同时实现或对同一内容做相反处理时，记录真冲突。

## 第二级：按证据排序

1. 优先保障人身安全、学科正确性和不可逆风险。
2. 其次依据输入元数据、官方学情 profile 和对应学科学段课标。
3. 再依据附录 A 的明文量规和 G1/G2 保真约束。
4. 再使用案例卡中不含答案与精确参数的可迁移方法。
5. 仅在证据强度与风险相当时参考多数意见。

## 第三级：决策

- 对能共存的建议合并，并列全部提案 ID 与来源角色。
- 对可通过缩小范围、增加支架或调整顺序化解的冲突，选择保真且风险更小的合并方案。
- 对必须二选一的冲突，接受证据更强、更安全、更保真的提案，并拒绝另一提案。
- 对仍无法解决且会实质改变用户意图的冲突，在 `CONFLICTS` 中写明双方证据并设置 `requires_user: true`；终稿中采用最小、可逆的临时处理。
