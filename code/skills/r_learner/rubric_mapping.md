# 技能：A/B/D 课型量规映射

按附录 A 区分常规课与 PBL 课，检查结构、可执行度和一致性。

## 课型与 A 结构完整性

- 以输入元数据声明的课型为准；保持输出课型不变。
- **常规课核心件只有**：教学目标；教学重难点；教学过程。教学过程应含情境导入、探究学习、例题巩固、课堂小结或同义环节。
- **常规课非核心件**：作业布置；板书设计。**推荐件**：学情分析；教材分析；教学反思。
- **PBL 核心件只有**：项目简介（背景与价值）；项目目标（含素养目标）；驱动性问题；任务链（项目—任务—活动分级且有衔接说明）。
- **PBL 非核心件**：每任务的总结反思；成果交流设计。
- **PBL 特别规定**：双轨评价设计缺失时，同时触发 A 缺件和 F6 = 0；不将双轨评价误列为上述四个 PBL 核心件之一。
- 对任一课型，缺任一核心件时 A 原始维度分封顶 2；先核对同义标题和实质内容，不凭标题字面自动判缺。
- 仅将包含本课概念、具体活动、数据或例题且非通用模板句的结构件视为有实质内容。

## B 内容丰富性

- **B1 环节展开度**：检查大多数环节或任务是否同时写明教师与学生活动。
- **B2 可执行颗粒度**：检查时间、组织形式与关键活动材料；不以篇幅代替可执行性。
- **B3 预设与应对**：要求探究/研讨主问题有绑定本课概念、预期答案要点或典型误区的预设。
- **B4 支架材料**：要求至少一件支架与主活动的数据或任务直接耦合。
- 对 PBL 将 B1—B4 应用于任务/活动单元，不另造与附录 A 不一致的计分项。

## D 内容一致性

- **D1**：独立核对每条目标的活动与检核证据；不采信“本环节落实目标”等自我声明。
- **D2**：为每个难点提供脚手架、变式或直观化突破。
- **D3**：要求前环节产出被后环节实质使用，不把过渡语当作衔接证据。
- **D4**：对常规课核对作业/板书与目标；对 PBL 核对成果交流/成果评价与项目目标。
- **D5**：以官方学情 profile 核对难度和开放度，并交 `r_literacy` 协同判断。
- 对 PBL 使用“项目目标—任务链—成果检核”视图；对常规课使用“目标—教学环节—课内/作业检核”视图。
